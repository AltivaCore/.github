<div align="center">

<h2>Organización de repositorios</h2>

</div>

<table>
  <tr>
    <td width="50%" valign="top">
      <h3>Repositorios públicos</h3>
      <p>
        Se utilizan para presentación institucional, documentación general, landing pages, componentes no sensibles y material técnico demostrativo.
      </p>
      <p>
        No contienen credenciales, datos reales de clientes, rutas internas, consultas productivas sensibles ni configuraciones privadas.
      </p>
    </td>
    <td width="50%" valign="top">
      <h3>Repositorios privados</h3>
      <p>
        Se reservan para productos en desarrollo, lógica de negocio, integraciones reales, automatizaciones productivas, consultas internas y documentación confidencial.
      </p>
      <p>
        El código operativo y las configuraciones reales se mantienen separados del contenido público.
      </p>
    </td>
  </tr>
</table>
